## Overview

This document outlines all images needed for the SimpliCares landing page, with sourcing recommendations and specifications.

## 1. Hero Section Images

### Primary Hero Image
**Filename:** `hero-caregiver.jpg` or `hero-caregiver.webp`
**Description:** Warm, professional image of a caregiver using a smartphone
**Specifications:**
- Resolution: 1920x1080 minimum
- Style: Warm lighting, authentic, diverse representation
- Elements: Caregiver (nurse/aide), smartphone visible, home setting or patient interaction

**Free Stock Sources:**
- [Unsplash](https://unsplash.com/s/photos/caregiver-phone) - Search "caregiver phone" or "home health aide"
- [Pexels](https://pexels.com) - Search "nurse mobile" or "healthcare worker phone"
- [Pixabay](https://pixabay.com) - Search "caregiver technology"

**Recommended Searches:**
- "home care nurse smartphone"
- "caregiver with elderly patient"
- "healthcare worker mobile app"
- "nurse using phone patient home"

### App Mockup Image
**Filename:** `app-mockup.png`
**Description:** SimpliCares app interface displayed on phone mockup
**Specifications:**
- Transparent background (PNG)
- iPhone or Android device frame
- Show key screen (dashboard or check-in)

**Tools to Create:**

## Image Optimization

### Before Adding Images

1. **Compress images:**
   - [Squoosh](https://squoosh.app) - Google's image optimizer
   - [TinyPNG](https://tinypng.com) - PNG/JPG compression
   - [SVGOMG](https://jakearchibald.github.io/svgomg/) - SVG optimizer

2. **Use modern formats:**
   - WebP for photos (better compression)
   - SVG for icons and logos (scalable)
   - PNG for screenshots with transparency

3. **Responsive images:**
   - Provide multiple sizes (1x, 2x for retina)
   - Use Next.js Image component for automatic optimization

### Next.js Image Usage
```jsx
import Image from 'next/image'

<Image
  src="/images/hero/hero-caregiver.webp"
  alt="Caregiver using SimpliCares app"
  width={600}
  height={400}
  priority // for above-the-fold images
/>
```
